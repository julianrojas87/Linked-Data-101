Linked Data 101 - A tutorial for Web developers

In the same way that websites being hosted on different servers can be linked to each other to allow users surf the Web and discover more content, Linked Data introduces a set of design principles for interconnecting data that allow applications to surf and reuse data on the Web. These principles open the door for developers to create richer and more innovative applications that combine data from heterogeneous and independent sources. However, it is common that developers who are not familiarized with the concepts and technologies related to Linked Data, struggle to find the proper tools that allow them unleash the full potential that Linked Data has to offer.

For that reason, in this tutorial we present some of the existing alternatives and tools that facilitate working with Linked Data in Web applications. We describe the basic concepts and some common tasks that Web developers face when creating applications that use Linked Data. Depending on your goal(s) and your prior knowledge about Linked Data, you can sequentially follow this tutorial for a comprehensive overview or dive directly into any of the following usage examples:

  • Working with JSON-LD data
  • Using ontologies in your UI

Basic concepts

What is Linked Data

The Web started as a collection of documents or websites, that could be accessed and uniquely identified by an URL, e.g. http://info.cern.ch/hypertext/WWW/TheProject.html â€” the first website ever. The information contained in these websites is normally presented in a human-friendly format and often links to other websites containing more related information. However machines struggle trying to understand the data contained in these websites as humans do. For this reason a set of principles were defined for structuring and publishing data on the Web. These are known as the Linked Data principles and they aim on giving access to machines to all the data available on the Web.

How does Linked Data look like

RDF

The Resource Description Framework (RDF) is a standard model for representing information on the Web. It is part of the W3C standard recommendations and it is considered within the Linked Data principles, as the standard model for representing data on the Web. RDF defines an abstract model that can be materialized through different serialization formats. Some of them are based on well-known syntaxes that are familiar to Web developers such as JSON-LD or RDF/XML. Regardless of the serialization format, RDF data remains consistent and is semantically equivalent from one format to another. This means that in theory, a Linked Data-based Web application should be capable of reusing and automatically understanding RDF data coming from different sources and in different formats. In practice, this requires data parsing processes for which several tools exist already.

The model defined by RDF allows to make statements about resources (e.g. documents, people, physical objects, abstract concepts, etc) in the form of triples. A triple expresses a relationship between two resources and it is composed by a subject, a predicate and an object (see figure 1).

Fig 1. An RDF triple.

Some examples of RDF triples are the following: 

                            
<Bob> <is a> <person>
<Bob> <is a friend of> <Alice>
<Bob> <is born on> <the 27th of June 1987>
<Bob> <is interested in> <the Mona Lisa>
<the Mona Lisa> <was created by> <Leonardo da Vinci>
